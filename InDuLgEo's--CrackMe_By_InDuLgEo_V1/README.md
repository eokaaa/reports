## Анализ

### Упаковка

DiE выявил три слоя упаковки: UPX и два неизвестных паковщика. После снятия двух внешних слоёв исполняемый код стал читаемым. UPX-слой оставлен намеренно — без него вся логика видна в открытом виде.

### Алгоритм проверки

Крякмис принимает два поля: `name` и `key`. В IDA Pro восстановлена логика проверки:

**Статический ключ (неосновное решение):**
```
name: <любое>
key:  AoRE2025
```
Захардкожен в теле функции проверки.

**Алгоритм кейгена (основное решение):**
```
key = "AORE-" + str(len(name) * 1337)
```
Длина введённого имени умножается на константу `1337`, результат подставляется после префикса `AORE-`.

Помимо этого в исходнике через XOR обфусцированы строки, выводимые в консоль — деобфусцированы статически.
