## Ход работы

### 1. Первичная разведка

Открыл бинарь в IDA. Так как с реверсом Rust-бинарников раньше не сталкивался, спросил у ИИ, где в таких бинарниках искать `main` — зашёл по подсказке, но ничего примечательного там не увидел.

Перешёл к строкам (`Strings`). Сначала проверил очевидные варианты — поиск `HTB{`/`}` напрямую по строкам ничего не дал (ожидаемо, учитывая заявленный уровень сложности). Дальше поискал по ключевым словам `password`, `incorrect`, `correct` — нашлась строка `"Correct one of us!!\n"`. Перешёл по перекрёстным ссылкам на неё и попал на код с приветствием и запросом ввода — судя по контексту, нужное место для дальнейшего анализа.

### 2. Регулярные выражения-подсказки

Рядом с сообщениями, ведущими к вводу, обнаружился набор регулярных выражений — судя по всему, зашитые в бинарь как часть проверки формата ответа:

```
^.{33}$
(?:^[\x48][\x54][\x42]).*
^.{3}(\x7b).*(\x7d)$
^[[:upper:]]{3}.[[:upper:]].{3}[[:upper:]].{3}[[:upper:]].{3}[[:upper:]].{4}[[:upper:]].{2}[[:upper:]].{3}[[:upper:]].{4}$
(?:.*\x5f.*)
(?:.[^0-9]*\d.*){5}
.{24}\x54.\x65.\x54.*
^.{4}[X-Z]\d._[A]\D\d.................[[:upper:]][n-x]{2}[n|c].$
.{11}_T[h|7]\d_[[:upper:]]\dn[a-h]_[O]\d_[[:alpha:]]{3}_.{5}
```

Смысл каждого — сузить формат итоговой строки (длина 33 символа, начинается с `HTB{`, заканчивается `}`, содержит подчёркивания, определённое распределение заглавных/строчных букв и цифр по позициям и т.д.). С помощью ИИ начал собирать из этих ограничений итоговую формулу, но на каком-то этапе прогресс застопорился — пробовал воссоздать общую формулу вручную, тоже без особого результата.

### 3. Функция `rega_town::check_input`

Предположил, что часть логики может быть не в регулярках, а прямо в коде — и что нужно поискать что-то ещё в бинаре. ИИ подтвердила предположение и указала на функцию `rega_town::check_input`, найденную в `main`. Внутри — константы, представляющие собой произведения кодов символов. Это дало понимание, в каких позициях финальной строки находятся какие буквы. Три слова из формулы благодаря этому совпали сразу, но их значения всё равно нужно было вычислить.

### 4. Восстановление символов по произведениям

**Случай 1** — два символа в произведении, один из них уже был известен (зафиксирован регуляркой/константой):

```
итог / известный_символ = искомый_символ
```

Решалось прямым делением.

**Случай 2** — сложнее: 4 символа в произведении, известны только диапазоны:
- 1-й — заглавная буква (`A`–`Z`)
- 2-й — цифра (`0`–`9`)
- 3-й — фиксированный символ `n`
- 4-й — строчная буква (`a`–`h`)

Перебирать вручную было бессмысленно — написал простой брутфорс-кейген на C++:

```cpp
#include <iostream>

int main() {
    int i = 41637750;

    for (int j = (int)'A'; j <= (int)'Z'; ++j)
    {
        for (int n = (int)'0'; n <= (int)'9'; ++n)
        {
            for (int c = (int)'a'; c <= (int)'h'; ++c)
            {
                int e = j * (int)'n' * n * c;
                if (e == i)
                    std::cout << j << ' ' << n << ' ' << c;
            }
        }
    }
}
```

Результат (коды символов): `75 49 110 103` → `K`, `1`, `n`, `g` → слово **`K1ng`**.

Остальные слова формулы были восстановлены с помощью ИИ по аналогичной логике (сопоставление регулярок и констант из `check_input`).

### 5. Финальная формула и флаг

```
^[\x48][\x54][\x42](\x7b)[X-Z]\d._[A]\D\d_T[h|7]\d_[[:upper:]]\dn[a-h]_[O]\d_\x54.\x65.\x54[[:alpha:]]{3}(\x7d)
```

**Флаг:** `HTB{Y0u_Ar3_Th3_K1ng_O7_The_Town}`

## Ограничения / что не входило в разбор

Бинарь ELF-формата, флаг получить динамически (просто введя предполагаемый ответ в запущенную программу и подобрав его) не было возможности — решение строилось полностью статически, через восстановление формулы из регулярных выражений и констант в `check_input`.

## Заключение

Первый опыт реверса Rust-бинарника. Основная сложность — в отсутствии опыта работы с регулярными выражениями: набор regex-ограничений в бинаре задавал формат флага, но самих значений не давал, и без сопоставления их с константами в `check_input` собрать формулу не получалось. Уровень заявлен как Medium, но после того как стало понятно, где искать (`check_input` + регулярки), сама задача оказалась достаточно прямолинейной — за исключением пары символов, для которых потребовался небольшой брутфорс-кейген.
